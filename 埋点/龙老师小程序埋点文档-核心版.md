# 龙老师小程序 - 核心埋点文档（精简版）

本文档在《龙老师小程序埋点文档》全量版基础上，**仅保留具备明确数据分析价值的埋点**，便于聚焦转化漏斗、核心功能使用、局端驾驶舱与产品迭代决策。格式仍遵循 **NewFrog 埋点规则**（url：`/事件类型/页面名称/事件名称`，小驼峰，参数驼峰）。

---

## 一、筛选原则说明

### 1.1 保留的埋点类型

| 类型 | 说明 | 典型用途 |
|------|------|----------|
| **转化漏斗** | 登录入口→点击登录→成功/失败；首页→各功能入口 | 注册转化率、失败原因分布、功能流量分配 |
| **核心行为与结果** | 拍照、识别成功/失败、加入错题本、录错题保存、擦除完成、AI 讲解 | 使用次数、使用人数、局端驾驶舱指标 |
| **关键业务结果** | 登录成功/失败、识别成功/失败、错题保存成功、擦除完成 | 成功率、完成率、异常监控 |
| **必要入口与出口** | 首页各功能入口、立即登录、设置/退出登录 | 路径分析、留存与流失 |

### 1.2 本版剔除的埋点（无分析价值或可由现有埋点推导）

- **协议/合规类**：用户协议、隐私政策、协议弹窗取消/前往阅读、设置页协议链接 → 对业务决策和驾驶舱无指标需求，不单独上报。
- **取消/返回类**：各类「取消」、返回、关闭详情 → 不改变业务状态，分析价值低；路径可由「入口 + 结果」反推。
- **重复或过细步骤**：获取验证码、学龄/地区/教育阶段每一步单独点击 → 保留「点击登录」+「登录成功/失败」+「确认学龄等信息」即可分析漏斗。
- **页内细粒度操作**：闪光灯、重拍、框选确认、上一题/下一题、切换结果标签、录错题内勾选/手动框题/科目弹窗确定取消、打印的预览/去查看、擦除内下一步/确定、填写答案上下题、更多、分享到朋友圈/好友拆开等 → 由「入口 + 最终结果」即可衡量功能使用，不逐一埋点。
- **管理类细粒度**：全选、标记已订正/未订正、删除取消 → 保留「删除」+「删除确认」即可统计管理行为。
- **部分页面曝光**：非核心页的 pageView 可后续按需加回；本版仅保留对漏斗或驾驶舱有直接作用的曝光（可选）。

### 1.3 全局参数：grade（年级）

**除登录页（teacherLongLoginPage）的事件外，所有事件均需携带 `grade#STRING` 参数**，记录用户当前年级。该参数用于驾驶舱「各年级活跃用户分布」等指标。研发实现时建议封装为公共参数自动注入，无需每个埋点手动传入。

### 1.4 URL 与参数规范（与全量版一致）

- **格式**：`/事件类型/页面名称/事件名称`，多词小驼峰。
- **页面名称（本版约定）**：**统一带 `teacherLong` 前缀**（如 `teacherLongLoginPage`、`teacherLongHomePage`），与系统内其他业务（如通用 teacherLongLoginPage）区分，避免重名；url 中全部使用带前缀的页面名。
- **事件类型**：**click** / **event**（结果、曝光） / **duration** / **time** / **other**。
- **root 为 click 时**，actionName 不以 click 开头（如 `phoneLogin` 而非 `clickPhoneLogin`）。
- **参数 key**：小驼峰；参数说明格式同全量版。

## 四、埋点与局端数据驾驶舱对应关系

与全量版第四节一致：驾驶舱按「使用功能、使用次数、使用人数、学科、时间、空间」聚合，本精简版埋点可完全支撑下表指标。

| 驾驶舱指标 | 含义 | 对应埋点 / 聚合规则 |
|------------|------|----------------------|
| 参与学生数 | 有使用记录的学生去重人数 | `loginSuccess` 按 userId 去重 |
| 学生 DAU | 当日登录去重 UV | `loginSuccess` 按日 distinct userId |
| 作业批改-次数/UV | 学生发起作业批改次数及人数 | `pageView(correctionResultPage)` count / distinct userId |
| 拍照答疑-次数/UV | 学生拍照答疑次数及人数 | `pageView(qaResultPage)` count / distinct userId |
| AI 答疑-次数/UV | 学生使用 AI 答疑（苏氏问答）次数及人数 | `pageViewSocraticQa` count / distinct userId（所有来源汇总，AI 答疑仅从搜索结果页进入） |
| 上传错题-次数/UV | 学生上传错题次数及人数 | `wrongSaveSuccess` count / distinct userId |
| 试卷擦除-次数/UV | 试卷擦除使用次数及人数 | `eraseComplete` count / distinct userId |
| AI 讲题-次数/UV | AI 讲题使用次数及人数 | `pageViewSocraticQa` count / distinct userId（进入苏氏问答页即视为使用 AI 讲题） |
| 错题本-使用人数 | 错题本功能使用人数 | `pageView(wrongBookPage)` distinct userId |
| 各年级活跃用户分布 | 按年级分组的去重用户 | 所有非登录事件携带 grade 参数，按 grade 分组 distinct userId |
| 热门功能 TOP5（学生端部分） | 使用人数 UV 最高的功能 | 上述各功能 distinct userId 排序取 TOP |
| 按学科分布 | 各学科使用量 | 带 `subject` 参数的事件按 `subject` 分组统计 |

**实施要求**：
1. 每条上报携带 **userId**，以便按用户去重。
2. 除登录事件外，每条上报携带 **grade** 参数（见 1.3 节），驾驶舱的年级分布依赖此字段。
