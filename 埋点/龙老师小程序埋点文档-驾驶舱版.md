# 龙老师小程序 - 核心埋点文档（驾驶舱版）

本文档仅保留**直接支撑数据驾驶舱指标**的埋点事件，以及必要的异常监控配对事件。格式遵循 **NewFrog 埋点规则**（url：`/事件类型/页面名称/事件名称`，小驼峰，参数驼峰）。

---

## 一、筛选原则

### 1.1 保留标准

| 类型 | 说明 |
|------|------|
| **驾驶舱 KPI 依赖** | 累计参与学生、学生 DAU、累计服务次数 |
| **驾驶舱模块数据源** | 右栏 AI 助学 6 项指标、年级分布、弹幕 |
| **成功/失败配对** | 与驾驶舱指标配对的失败事件，用于识别成功率监控 |

### 1.2 剔除内容（与核心版对比）

- **首页入口点击**（enterAiCorrection / enterPhotoQa / enterWrongBook 等）→ 驾驶舱只统计完成结果，不统计入口流量
- **登录中间步骤**（点击登录、确认学龄等信息）→ 登录成功+失败已覆盖漏斗
- **我的 / 设置 / 退出登录** → 驾驶舱无对应指标
- **拍照中间步骤**（快门、相册）→ 识别成功率 = success / (success + fail)
- **搜索结果页操作**（加入错题本点击、录错题点击、AI 讲解点击）→ 仅保留成功结果事件
- **错题本管理**（去添加、切换科目、添加入口、打印、删除）→ 驾驶舱仅需页面曝光统计使用人数
- **历史记录全部事件** → 驾驶舱无对应指标
- **我的试卷全部事件** → 驾驶舱无对应指标
- **苏氏问答交互**（语音、相机、键盘）→ 驾驶舱仅需曝光和播放
- **时长事件**（playDuration / sessionDuration / stayDuration）→ 「累计学习时长」已从驾驶舱 KPI 移除
- **试卷擦除中间步骤**（擦除按钮、拍照、分享）→ 驾驶舱仅需擦除完成

### 1.3 规范

- **URL 格式**：`/事件类型/页面名称/事件名称`，多词小驼峰
- **页面名称**：统一 `teacherLong` 前缀
- **事件类型**：**event**（结果、曝光） / **click**（用户主动操作）
- **每条上报自动携带**：userId（用户编号）、grade 快照（年级，研发自动附加）

## 二、核心埋点清单（共 11 条）

### 2.1 登录（teacherLongLoginPage）

| 序号 | url | 事件名称 | 事件说明 | 触发逻辑 | 参数 | 参数说明 |
|------|-----|----------|----------|----------|------|----------|
| 1 | /event/teacherLongLoginPage/loginSuccess | loginSuccess | 登录成功 | 校验通过、即将跳转首页前上报 | loginType#STRING | loginType：登录方式__phone-手机验证码 |
| 2 | /event/teacherLongLoginPage/loginFail | loginFail | 登录失败 | 校验失败时上报 | reason#STRING | reason：失败原因__code_error-验证码错误##not_registered-未注册 等 |

> **驾驶舱用途**：`loginSuccess` → KPI「累计参与学生」（distinct userId）、「学生 DAU」（按日 distinct userId）、年级分布（按 grade 快照分组）。`loginFail` 用于登录异常监控。

---

### 2.2 拍照承载页（teacherLongCameraPage）

通过参数 `mode` 区分功能：correction（作业批改）/ qa（拍照答疑）/ wrong（上传错题）/ erase（试卷擦除）。

| 序号 | url | 事件名称 | 事件说明 | 触发逻辑 | 参数 | 参数说明 |
|------|-----|----------|----------|----------|------|----------|
| 1 | /event/teacherLongCameraPage/recognitionSuccess | recognitionSuccess | 识别成功 | 识别完成且成功时上报 | mode#STRING | mode：correction##qa##wrong##erase |
| 2 | /event/teacherLongCameraPage/recognitionFail | recognitionFail | 识别失败 | 识别失败时上报 | mode#STRING | 同上 |

> **驾驶舱用途**：`recognitionSuccess(mode=correction)` → 作业批改次数/UV；`recognitionSuccess(mode=qa)` → 拍照答疑次数/UV。`recognitionFail` 用于识别成功率监控。弹幕「完成了AI作业批改」「使用了拍照答疑」分别取 mode=correction / qa。

---

### 2.3 作业批改结果页（teacherLongCorrectionResultPage）

结果页可从拍照承载页或历史记录进入，通过 `source` 参数区分来源：camera（拍照进入）/ history（历史记录进入）。

| 序号 | url | 事件名称 | 事件说明 | 触发逻辑 | 参数 | 参数说明 |
|------|-----|----------|----------|----------|------|----------|
| 1 | /click/teacherLongCorrectionResultPage/playVideo | playVideo | 播放视频讲解 | 用户点击视频讲解播放时上报 | source#STRING,questionId#STRING | source：camera-拍照进入##history-历史记录进入；questionId：题目 id |
| 2 | /event/teacherLongCorrectionResultPage/addToWrongSuccess | addToWrongSuccess | 加入错题本成功 | 加入成功、Toast 展示时上报 | source#STRING,questionId#STRING | source：camera##history；questionId：题目 id |

> **驾驶舱用途**：`playVideo` count / distinct userId → 视频讲解播放次数/UV。`addToWrongSuccess` count → 错题本累计收录题数（作业批改侧）。

### 2.5 上传错题结果页（teacherLongUploadWrongResultPage）

| 序号 | url | 事件名称 | 事件说明 | 触发逻辑 | 参数 | 参数说明 |
|------|-----|----------|----------|----------|------|----------|

## 三、驾驶舱指标完整映射

下表列出驾驶舱（学生端部分）每项指标与埋点事件的对应关系。

| 驾驶舱指标 | 含义 | 对应埋点 / 聚合规则 |
|------------|------|----------------------|
| 累计参与学生 | 登录过的学生去重人数 | `loginSuccess` distinct userId |
| 学生 DAU | 当日登录去重 UV | `loginSuccess` 按日 distinct userId |
| 作业批改-次数/UV | 作业批改使用次数及人数 | `recognitionSuccess(mode=correction)` count / distinct userId |
| 拍照答疑-次数/UV | 拍照答疑使用次数及人数 | `recognitionSuccess(mode=qa)` count / distinct userId |
| AI 答疑-次数/UV | AI 答疑使用次数及人数 | `pageViewSocraticQa` count / distinct userId |
| 上传错题-次数/UV | 上传错题使用次数及人数 | `wrongSaveSuccess` count / distinct userId |
| 试卷擦除-次数/UV | 试卷擦除使用次数及人数 | `eraseComplete` count / distinct userId |
| 视频讲解-播放次数/UV | 结果页视频讲解播放次数及人数 | `playVideo(resultPage)` count / distinct userId |
| AI 讲题-次数/UV | 苏氏问答页 AI 讲题播放次数及人数 | `play(socraticQaPage)` count / distinct userId |
| 错题本-使用人数 | 进入错题本的去重人数 | `pageView(wrongBookPage)` distinct userId |
| 错题本-累计收录题数 | 错题本中累计录入的题目数 | `addToWrongSuccess` count + SUM(`wrongSaveSuccess`.count) |
| 试卷打印-套数 | 打印试卷的累计次数 | `export(printWrongPage)` count 次 |
| 累计服务次数 | 所有功能使用次数汇总 | SUM(作业批改 + 拍照答疑 + AI答疑 + 上传错题 + 试卷擦除 + 视频讲解 + AI讲题 + 错题本使用 + 教师端教学动画 + AI命题 + AI组题) |
| 各年级活跃用户分布 | 按年级分组的去重用户 | 上述各事件按 grade 快照分组 distinct userId |

## 六、实施要求

1. **userId**：每条上报必须携带用户编号，驾驶舱所有 UV 指标依赖此字段。
2. **grade 快照**：研发在事件上报时自动附加当前用户的年级字段，驾驶舱的年级分布和弹幕均依赖此字段。
3. **phone 后四位**：弹幕显示需要，由研发从用户资料中截取后四位自动附加，不作为事件参数定义。
4. **测试重点**：按「登录成功 → 识别成功 → 加入错题本成功 / 录错题保存成功 → 擦除完成 → 苏氏问答曝光 → 播放」链路验证。
5. **扩展说明**：本版为驾驶舱专用最小集（11 条事件）。若后续需要漏斗分析（首页入口→功能完成转化率）、用户路径分析、管理行为分析等，可从《龙老师小程序埋点文档-核心版》中按需补充。
