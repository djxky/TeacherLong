1. 页面概述

功能说明

历史记录页汇总用户使用 AI 工具产生的所有结果记录，按功能类型分 Tab 展示（作业批改、拍照答疑、错因诊断），支持按学科筛选，点击卡片可回看历史结果。

定位是"学习轨迹回顾"——让学生和家长能随时查看之前的批改结果、答疑记录和诊断报告，不用每次重新拍照。

功能边界：本页面只负责展示和导航到历史记录，不支持编辑或重新分析。

2. 顶部 Tab 切换

交互流程

1. 点击"作业批改/拍照答疑/错因诊断"Tab → 切换到对应类型的记录列表

2. 切换 Tab 时学科筛选重置为"全部"

业务规则

  • 三个 Tab 固定展示，不因无数据而隐藏
  • 当前激活 Tab 有下划线指示器
  • 默认进入"作业批改"Tab

3. 学科筛选

交互流程

1. 顶部横向滚动的筛选药丸按钮：全部 (N) / 数学 (N) / 英语 (N) / …

2. 点击筛选按钮 → 列表按学科过滤 → 筛选按钮高亮

业务规则

  • 筛选项动态生成：根据当前 Tab 下有数据的学科自动列出，无数据的学科不展示
  • 每个筛选项后面显示记录数
  • 默认选中"全部"

数据说明

学科从记录数据中自动提取去重，不依赖后端配置

4. 记录卡片列表

功能说明

按日期分组展示历史记录，每条记录以卡片形式呈现，包含作业缩略图和摘要信息。

交互流程

1. 上下滚动浏览记录，按日期分组（今天 / 昨天 / 具体日期）

2. 点击卡片 → 跳转到对应的结果页面

Tab 类型卡片展示跳转目标
作业批改作业缩略图 + "共N题"批改结果页
拍照答疑题目缩略图 + "共N题"答疑结果页
错因诊断作业缩略图 + "N题错误"错因诊断页

业务规则

  • 双列网格布局（2 列）
  • 同一日期的记录归入同一组
  • 日期显示规则：今天 → "今天"，昨天 → "昨天"，更早 → "X月X日"
  • 列表滚动到底部显示"已到底部"提示

数据说明

每条记录包含：ID、日期、时间、学科、缩略图 URL、题目数量。错因诊断额外包含错题数量。
数据按时间倒序排列，最新的在最前。

异常场景

场景处理方式
当前 Tab 无任何记录居中灰色时钟图标 48px +「还没有使用记录」（14px #333）+「拍照解题、作业批改、错因分析的记录都会出现在这里」（12px #999）+ 按钮「去首页看看」→ 跳转首页
筛选后无数据居中灰色搜索图标 48px +「暂无相关记录」（14px #333）+「当前筛选条件下没有找到记录」（12px #999），无引导按钮
缩略图加载失败显示灰色占位图（与卡片等宽等高），不影响卡片点击
列表加载中双列骨架屏占位（与卡片尺寸对齐），每列 2 个骨架卡片
列表加载失败页面中部显示「加载失败」+ 重试按钮，点击重新请求数据

5. 底部管理按钮

功能说明

底部固定"管理"按钮，用于后续支持批量删除、导出等操作。当前版本仅展示占位。

业务规则

  • 点击"管理"按钮 → Toast 提示"管理功能（即将上线）"

6. 导航

交互流程

1. 返回：点击左上角返回按钮 → 回到进入时的首页版本

业务规则

  • 返回首页时需记住用户之前使用的首页版本，跳转到对应版本
  • 无浏览历史时兜底返回默认首页
