## 1. AI 龙老师对话入口卡片

### 功能说明

首屏核心模块，承载 AI 对话的三种入口（文字 / 语音 / 拍照）。定位是"AI 即老师"的产品心智——用户第一眼看到的不是传统工具列表，而是一个可以对话的 AI 老师。

选择把 AI 卡片放在首屏最顶部而非像 v1 那样和工具平级，是因为 v3 的核心差异就是"AI 优先"。v1 是工具导向，v3 是对话导向。

功能边界：本卡片只负责引导用户进入对话页，不在首页内完成任何 AI 交互。

### 交互流程

1. **文字入口**：用户点击输入条任意区域 → `sessionStorage` 存入 `aiSource=home-focus` → 跳转 `对话.html` → 对话页读取 source 后自动 focus 输入框弹出键盘
2. **语音入口**：用户点击麦克风图标 → 弹出语音浮层（阻止事件冒泡，不触发输入条跳转） → 模拟识别 2 秒 → 显示识别文字"这道数学题怎么解" → 1 秒后 `sessionStorage` 存入 `aiQuestion` + `aiSource=home-voice` → 跳转 `对话.html` → 对话页自动发送该文字
3. **拍照入口**：用户点击相机图标 → 直接跳转 `作业/作业批改.html`（默认批改模式） → 拍照页底部可切换搜题/录错题/错因诊断模式

#### 为什么拍照不进对话页

拍照页底部已有模式切换入口（批改/搜题/录错题/错因诊断），用户无需先跟 AI 对话再选工具。如果用户想"拍照后跟 AI 聊"，可以在对话页内使用相机按钮。

### 业务规则

- 输入条 placeholder 固定文案 `给 AI 龙老师发消息`，品牌导向而非功能描述
- 副标题轮播每 3 秒切换一条，淡入淡出过渡，共 8 条（功能引导与百科问题交替）
- 语音识别结果 demo 固定为"这道数学题怎么解"，不调用真实语音 API

### 数据说明

> **副标题轮播内容**：前端硬编码数组，8 条内容交替展示功能引导（拍一拍作业秒批改）和百科问题（为什么天空是蓝色的），后续可改为后端下发支持个性化推荐。

> **sessionStorage 键值**：`aiQuestion`（用户问题文字）、`aiSource`（来源标识：home-focus/home-voice/home-input/home-prompt）、`aiImage`（base64 图片），页面初始化时写入，对话页消费后立即清除。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 语音识别过程中用户点击"取消" | 关闭浮层，不跳转，不存 sessionStorage |
| 语音识别过程中用户点击浮层外区域 | 同上，关闭浮层 |
| sessionStorage 被浏览器禁用 | 跳转对话页但无法传递上下文，显示空态，用户需手动输入（当前未做兜底提示） |

### 关联模块

- → `对话.html`：文字和语音入口的目标页，通过 sessionStorage 传递上下文
- → `作业/作业批改.html`：拍照入口的目标页
- ← 对话页返回时通过 `history.back()` 回到本页

### 不做（本期）

- 真实语音识别 API 对接（demo 为模拟）
- 个性化问候（如"下午好，小明同学"后端下发）——当前固定"Hi，我是 AI 龙老师"
- 输入条支持直接打字（当前点击即跳转对话页）

## 2. 常用工具区

### 功能说明

首屏第二个模块，紧接 AI 卡片下方。采用"双大卡 + 8 宫格"的层级布局，将 10 个工具按使用频率分两级展示。

选择这种布局而非 v1 的全平铺宫格，是为了制造视觉重心——作业批改和拍照搜题是最核心的高频工具，用大卡片突出；其余工具用小图标承载，避免首屏信息过载。

功能边界：工具区仅提供跳转入口，不承载任何工具的执行逻辑。

### 交互流程

1. **主功能大卡**：用户点击"作业批改" → 跳转 `作业/作业批改.html?mode=correction`；用户点击"拍照搜题" → 跳转 `作业/作业批改.html?mode=qa`
2. **8 宫格工具**：
   - 错题本 → `错题本/错题本.html`（直接进入错题列表）
   - 录错题 → `作业/作业批改.html?mode=wrong`（进入拍照页，模式为录错题）
   - 错因诊断 → `作业/作业批改.html?mode=diagnosis`（进入拍照页，模式为错因诊断）
   - 试卷擦除 → `试卷/试卷擦除.html`
   - 试卷下载 → `试卷/试卷下载.html`
   - 试卷讲解 → `试卷讲解/试卷讲解.html`
   - 小升初真题 → `试卷/试卷列表.html?type=primary`
   - 中考真题 → `试卷/试卷列表.html?type=middle`

### 业务规则

- 所有工具直接跳转功能页面，不经过 AI 对话页
- 用户通过 AI 对话页也可触达这些工具（AI 返回功能卡片），首页提供直达入口降低操作路径
- 工具入口无权限控制，所有用户可见

### 数据说明

> 工具列表为前端硬编码，不依赖后端接口。图标使用 SVG 内联。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 目标页面加载失败 | 浏览器默认 404（当前无自定义错误页） |
| 某个工具需下线 | 前端删除对应 DOM 节点，无动态控制机制 |

### 关联模块

- → `作业/作业批改.html`：作业批改、拍照搜题、录错题、错因诊断共用此拍照页，通过 `mode` 参数区分
- → 各试卷/错题本页面：独立页面

### 不做（本期）

- 工具排序/隐藏的后端配置化
- 各工具的使用次数角标
- 工具分组折叠

## 3. 视频课堂

### 功能说明

内容区域第一个模块，展示三种视频课程体系。采用一大两小的卡片布局，大卡"新课标重点讲"是重磅推荐内容，两个小卡"校内同步课"和"拓展提升课"为常规内容入口。

### 交互流程

1. 用户点击"新课标重点讲"大卡 → 跳转 `视频/知识点讲解.html?cat=new-standard`
2. 用户点击"校内同步课"小卡 → 跳转 `视频/知识点讲解.html?cat=sync-course`
3. 用户点击"拓展提升课"小卡 → 跳转 `视频/知识点讲解.html?cat=special-skills`

### 业务规则

- 大卡带"重磅推荐"角标，小卡无角标
- 大卡底部有学科标签（语文/数学/英语），仅展示不可筛选

### 数据说明

> 课程标题、描述、学科标签均为前端硬编码。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 视频资源不存在 | 跳转到知识点讲解页面后，由该页面自行处理（当前为 mock 页面） |

### 关联模块

- → `视频/知识点讲解.html`：通过 `cat` 参数区分课程类别

### 不做（本期）

- 课程进度展示（如"已学 30%"）
- 视频课程的收藏/点赞
- 按年级筛选视频课程

## 4. 专项大招课

### 功能说明

按学科分类的专项课程卡片区域，支持按年级筛选。覆盖小学（1-6 年级）和初中（7-9 年级）两个学段，语文和英语两个学科。

### 交互流程

1. 用户点击右上角年级选择器 → 弹出 9 宫格年级选择面板（一年级～九年级），当前已选年级高亮
2. 用户选择年级 → 面板关闭 → 按钮文字更新为所选年级 → 课程列表自动切换（1-6 年级显示小学课程，7-9 年级显示初中课程）
3. 用户点击任意课程卡片 → 跳转 `视频/知识点讲解.html`

### 业务规则

- 默认年级读取 `sessionStorage.gradeClass`（全局年级设置），无则默认"七年级"
- 年级选择结果写入 `sessionStorage.zkActiveGrade`，刷新页面后保持
- 小学课程 5 门：语文古诗名篇、语文看图写话、语文阅读理解、英语语法提升、英语阅读专项
- 初中课程 5 门：语文中文阅读、语文中文写作、语文名著专项、英语语法专项、英语题型技巧

### 数据说明

> **课程数据**：前端 `ZK` 对象，按 `primary`（小学）和 `middle`（初中）分组。每门课含 `title`（课程名）和 `color`（左侧竖条颜色，语文=绿色，英语=蓝色）。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 点击年级弹窗外区域 | 弹窗关闭，年级不变 |
| sessionStorage 中的年级值不在 1-9 范围 | 兜底使用默认值"七年级" |

### 关联模块

- → `视频/知识点讲解.html`：课程详情
- ↔ `sessionStorage.gradeClass`：全局年级设置（与精品试卷共享）

### 不做（本期）

- 数学/科学学科的课程
- 课程节数、时长展示
- 课程完成进度

## 5. 精品试卷

### 功能说明

首页底部的试卷推荐区域，提供三级联动筛选（学科→年级学期→类型），覆盖一年级到九年级所有学段。筛选能力与 v1 完全对等。

### 交互流程

1. **一级筛选（学科 Tab）**：用户点击"语文/数学/英语/科学" → 切换学科 → 重置三级筛选为"热门推荐" → 刷新试卷列表
2. **二级筛选（年级+学期）**：用户点击年级按钮 → 弹出面板，上方为"上/下学期"切换，下方为 1-9 年级网格 → 选择后面板关闭 → 按钮文字更新 → 刷新试卷列表
3. **三级筛选（类型）**：用户点击"热门推荐/小升初真题/期末试题/期中试题/单元测试" → 切换筛选条件 → 刷新试卷列表
4. **试卷点击**：点击试卷行 → 跳转 `试卷/试卷详情.html?id=XXX`
5. **查看全部**：点击底部"查看全部" → 跳转 `试卷/试卷下载.html`

### 业务规则

- 学科切换时，年级和学期保持上次选择，类型重置为"热门推荐"
- 年级/学期切换时，类型保持上次选择
- 默认值：学科=语文、年级=七年级、学期=上、类型=热门推荐
- 每个筛选组合最多展示 10 条试卷
- 试卷角标：支持"推荐"（紫色）和"精选"（绿色）两种

### 数据说明

> **试卷数据**：前端 `PAPER_DATA` 对象，按 `[学科][年级+学期+类型]` 三维索引。每条试卷含 `title`（标题）、`dl`（下载量，如"2.3万"）、`tag`（角标类型，recommend/pick/空）。
>
> **年级数据**：`GRADES` 数组（一年级～九年级）。
>
> **学期数据**：`SEMESTERS` 数组（上、下）。
>
> 所有数据为本地 mock，不依赖后端。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 某个筛选组合无数据 | 试卷列表为空，不展示任何内容（当前无空状态提示） |
| 筛选按钮在渐变背景上不可见 | 使用白色实底背景 + 灰色边框确保可见性（已修复） |
| 点击年级弹窗外区域 | 弹窗关闭 |

### 关联模块

- → `试卷/试卷详情.html`：试卷详情
- → `试卷/试卷下载.html`："查看全部"入口
- ↔ `sessionStorage.activePaperSubj` 等：筛选状态持久化

### 不做（本期）

- 试卷收藏/下载计数实时更新
- 试卷搜索功能
- 按学校/地区筛选
- 空数据状态的引导提示

## 6. 页面导航与全局行为

### 功能说明

包括顶部导航栏、底部版本切换、页面入场动画、以及跨页面返回逻辑。

### 交互流程

1. **顶栏"我的"Tab**：点击 → 跳转 `我的.html`
2. **顶栏历史记录图标**：点击 → 跳转 `历史记录.html`
3. **版本切换条**（仅 demo 用）：点击 v1/v2/v3 切换首页方案
4. **模式切换按钮**（页面右上角）：点击"过稿" → 弹出产品稿浮窗；点击"演示" → 关闭浮窗

### 业务规则

- 页面加载时写入 `sessionStorage.homeVersion=v3`
- 所有二级页面返回时优先 `history.back()`，兜底读取 `sessionStorage.homeVersion` 决定返回 v1 或 v3
- 入场动画：所有模块使用 `fadeUp` 动画，按延迟分组（d1→d2→d3→d5），动画曲线 `cubic-bezier(0.16, 1, 0.3, 1)`
- 支持 URL 参数 `?mode=review` 直接进入过稿模式

### 数据说明

> 导航栏高度 44px，右侧预留小程序胶囊按钮安全距离（`--app-capsule-clearance`）。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 二级页面直接刷新（无 history） | `history.back()` 无效时，读取 `sessionStorage.homeVersion` 跳转对应首页 |
| `sessionStorage.homeVersion` 缺失 | 兜底跳转 `首页.html`（v1） |

### 关联模块

- → `我的.html`：个人中心
- → `历史记录.html`：AI 对话历史
- ↔ 所有二级页面：通过 sessionStorage 维护返回路径

### 不做（本期）

- 版本切换条上线时移除（仅 demo 用）
- 底部 Tab 导航（当前仅顶栏）
- 页面切换过渡动画（当前为硬跳转）
