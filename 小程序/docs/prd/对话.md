# 对话页 产品稿

> 版本：v1.0 · 2026-04-14
> 页面文件：`小程序/对话.html`

## 1. AI 意图路由（routeReply）

### 功能说明

这是对话页的核心逻辑引擎——AI 接收用户输入后，按优先级匹配意图，决定返回文字回复还是工具功能卡片。选择用前端规则路由而非后端 NLU，是因为 demo 阶段需要确定性的行为来演示产品逻辑，避免 AI 大模型的随机性干扰演示。

功能边界：只做意图匹配和回复分发，不做真实的 AI 推理。

### 交互流程

AI 按以下优先级逐层匹配，命中即停止：

1. **追问检测**（最高优先级）：如果上一轮 AI 返回了功能卡片（`lastCardType` 非空），且用户输入包含追问关键词（"第X题"、"为什么"、"怎么错"、"讲一下"等） → 进入追问处理（见模块 3）
2. **加入错题本**：同时包含（加入/存/收录/放到/添加）+  错题 → 若有上下文（上一轮为批改结果），回复"已将 3 道错题加入错题本" + 错题本卡片；若无上下文，提示"请先拍照批改作业"
3. **工具意图匹配**：
   - 批改（关键词：批改、检查作业） → 批改报告卡片
   - 试卷擦除（关键词：擦除、还原、空白） → 试卷擦除卡片
   - 错因诊断（关键词：错因、诊断、薄弱、分析） → 错因报告卡片
   - 录入错题（关键词：错题 + 上传/录入） → 录入卡片
   - 查看错题本（关键词：错题本、错题） → 错题本卡片
   - 搜题（关键词：搜题 / 图片+这道题/怎么做/看看） → 搜题卡片 + 苏格拉底引导
4. **图片消息**（未匹配工具）：用户发送了图片但关键词未命中 → AI 回复"让我看看这道题…" → 进入苏格拉底引导
5. **百科知识库**：匹配 `knowledgeQA` 中的 9 组预设问答 → 返回详细知识回答
6. **自由问答**（兜底）：匹配通用关键词（知识点/练习/出题/讲解/不懂/不会）返回引导回复；均未命中则返回通用引导

### 业务规则

- 关键词匹配不区分大小写，使用 `indexOf` 简单匹配
- 功能卡片上的跳转链接带 `show=result` 参数，直接跳转工具结果页面（跳过拍照步骤）
- 每次返回卡片后，将卡片类型写入 `lastCardType`（用于追问检测）
- 纯文字回复不更新 `lastCardType`

### 数据说明

> **关键词库**：前端硬编码在 `routeReply` 函数中。
>
> **百科知识库**（`knowledgeQA`）：9 组 QA，覆盖天空蓝色、恐龙灭绝、人体骨头、光速、地球大小、月球、下雨、太阳等主题。每条回答使用 emoji 分段，包含数据和比喻，适合中小学生阅读。
>
> **功能卡片结构**：图标 + 标题 + 副标题 + 预览行（✓/✗/📊） + "点击查看"跳转按钮。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 用户输入为空 | 发送按钮为 disabled 态，不触发路由 |
| 用户输入只有空格/换行 | 同上，trim 后判空 |
| 多个工具关键词同时命中 | 按优先级匹配第一个命中的（批改 > 擦除 > 错因 > 录错题 > 搜题） |
| 百科问答内容过长 | 前端固定内容，不会超长（已控制在 200 字以内） |

### 关联模块

- → 各工具结果页：功能卡片点击跳转
- ← 首页：接收 sessionStorage 中的用户意图
- ↔ 苏格拉底引导（模块 2）：搜题意图触发
- ↔ 追问处理（模块 3）：功能卡片后的追问

### 不做（本期）

- 真实 NLU 意图识别（当前为关键词规则）
- 多轮对话上下文理解（仅支持一轮追问）
- 工具优先级的动态调整

## 2. 苏格拉底式引导（doSocratic）

### 功能说明

当 AI 识别到搜题意图时，不直接给出答案，而是通过 5 步引导对话帮助学生自主思考。这是产品的核心教育理念——"教会思考方法比给答案更重要"。

选择固定 5 步而非动态步数，是因为 demo 需要可预测的演示流程，且教育心理学中"5 步引导"是经典的脚手架教学法。

### 交互流程

| 步骤 | AI 提问 | 目的 |
|------|---------|------|
| 1 | "你先看看这道题，它考的是哪个知识点？" | 帮助学生定位知识点 |
| 2 | "那你觉得解这道题的第一步应该做什么？" | 引导思考解题思路 |
| 3 | "试试把已知条件列出来，找找它们之间的关系" | 培养条件分析能力 |
| 4 | "你的方向对了！继续算下去，写出你的过程" | 鼓励自主完成 |
| 5 | "太棒了！你自己想出来的。要不要再练一道？" | 正向反馈 + 引导练习 |

1. 用户发送搜题类问题 → AI 返回搜题卡片 + 第 1 步引导
2. 用户回复任意内容 → AI 返回第 2 步引导
3. 重复直到第 5 步 → 引导结束，`socraticStep` 重置为 0

### 业务规则

- 引导过程中用户的**每次回复都推进到下一步**，不做内容分析
- 完成 5 步后停止引导，后续回复进入正常路由
- 引导过程中用户发送新的工具类关键词（如"帮我批改"），会中断引导进入新的路由

### 数据说明

> **引导步骤**：前端硬编码 `SOCRATIC_STEPS` 数组（5 条），`socraticStep` 变量追踪当前步骤。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 用户连续发送多条消息 | 每条都推进一步，可能快速跳完 5 步 |
| 引导过程中刷新页面 | `socraticStep` 重置为 0，引导从头开始（状态未持久化） |
| 引导过程中用户发送新工具意图 | 中断引导，进入新的路由 |

### 关联模块

- ← AI 意图路由：搜题意图触发引导
- ↔ 对话区：引导内容以普通文字气泡形式展示

### 不做（本期）

- 根据学生回答内容智能调整引导方向
- 引导过程中的正确率评估
- 引导完成后自动出练习题

## 3. 追问处理（handleFollowUp）

### 功能说明

当用户在 AI 返回功能卡片后提出追问时，AI 根据上一轮卡片类型给出上下文相关的具体回复。这解决了用户看到结果后的自然追问需求——比如看到批改结果后问"第 2 题为什么错"。

### 交互流程

| 上下文（lastCardType） | 追问类型 | AI 回复 |
|--------|----------|---------|
| correction（批改） | "第X题为什么错" | 该题的题目、错误答案、正确答案和错因分析 |
| correction（批改） | "为什么错/错在哪" | 汇总 3 道错题的不同错因分析 |
| diagnosis（错因诊断） | "计算类" | 细分计算类错误的 3 个子类型和占比 |
| diagnosis（错因诊断） | "出题/练习" | 返回 3 道针对薄弱点的练习题 |
| search（搜题） | 任意追问 | 继续苏格拉底引导（跳转到模块 2） |

### 业务规则

- 追问检测关键词：第X题、为什么、怎么错、错在、讲一下、不懂、不会、再看看、详细
- 仅支持**一轮追问**——追问后 `lastCardType` 不更新，再次追问进入正常路由
- 追问回复为纯文字气泡，不返回新的功能卡片

### 数据说明

> 追问回复内容为前端硬编码，模拟真实的 AI 分析结果（如具体题目、错因分析数据）。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 用户追问但 lastCardType 已被清除 | 进入正常路由，不进入追问处理 |
| 追问内容同时命中追问关键词和工具关键词 | 追问优先级更高，进入追问处理 |

### 关联模块

- ← AI 意图路由：追问检测为最高优先级
- ← 功能卡片：`lastCardType` 由卡片返回时设置

### 不做（本期）

- 多轮追问（当前仅支持一轮）
- 追问后返回新的功能卡片
- 追问内容的语义理解

## 4. 对话区 UI

### 功能说明

对话区承载所有消息的展示，包括用户消息、AI 文字回复、AI 功能卡片、输入中指示器。参考豆包和 Gemini 的对话 UI 设计。

### 交互流程

1. **用户发送文字**：右侧紫色气泡，文字自适应宽度
2. **用户发送图片**：右侧紫色气泡 + 图片缩略图（max 200px），图片上方可附带文字
3. **AI 文字回复**：延迟 800ms 后显示（模拟思考），先展示三点跳动的输入指示器，然后替换为文字气泡
4. **AI 功能卡片**：延迟 800ms 后显示，卡片内包含图标+标题+描述+预览行+跳转按钮
5. **点击功能卡片按钮**：跳转至对应工具结果页（带 `show=result` 参数）

### 业务规则

- 每条新消息后自动滚动到底部（`scrollTop = scrollHeight`）
- 用户消息最大宽度 80%，AI 消息最大宽度 85%
- 功能卡片的预览行最多 3 行，使用 ✓/✗/📊 前缀区分
- 输入指示器动画使用 CSS `blink` keyframes

### 数据说明

> 对话消息为实时 DOM 插入，不持久化。刷新页面后对话记录丢失。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 消息内容超长（数百字） | 文字气泡自适应高度，不截断 |
| 快速连续发送多条消息 | 每条消息独立处理，按顺序追加到对话区 |
| 对话内容超出视口 | 对话区可滚动，新消息自动滚动到底部 |
| 刷新页面 | 对话记录丢失，回到空态 |

### 关联模块

- ← AI 意图路由：决定展示文字回复还是功能卡片
- → 各工具结果页：功能卡片点击跳转

### 不做（本期）

- 对话记录持久化（本地存储或云端同步）
- 消息撤回/删除
- 流式打字机效果（当前为整段输出）
- 消息时间戳展示
- AI 回复中的 Markdown 渲染

## 5. 输入栏

### 功能说明

底部固定输入栏，包含四个元素：相机按钮、文字输入框、语音按钮、发送按钮。是用户与 AI 交互的唯一输入入口。

### 交互流程

1. **文字发送**：用户在 textarea 输入内容 → 发送按钮从灰色变为紫色可点击态 → 点击发送或按回车 → 创建用户气泡 → 触发 `routeReply` → 输入框清空 → 发送按钮恢复灰色
2. **图片发送**：用户点击相机按钮 → 触发隐藏的 `<input type="file">` → 选择图片 → 图片缩略图预览出现在输入框上方 → 用户可点击 ✕ 移除图片 → 用户输入文字（可选）后点击发送 → 创建含图片的用户气泡
3. **语音发送**：用户点击语音按钮 → 弹出语音浮层 → 模拟识别 → 自动发送识别文字

### 业务规则

- 发送按钮状态：输入内容非空 OR 有待发送图片时可点击，否则为 disabled
- textarea 自适应高度：最小 40px，最大 100px
- 回车发送：`Enter` 键发送，`Shift+Enter` 换行
- 图片预览尺寸：48×48 圆角缩略图
- 图片格式：接受浏览器支持的所有图片格式（`accept="image/*"`）

### 数据说明

> 图片选择后使用 `FileReader.readAsDataURL` 转为 base64 存储在 `pendingImage` 变量中。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| 选择超大图片（>10MB） | 无限制，直接使用（demo 不做大小校验） |
| 选择非图片文件 | `accept="image/*"` 在文件选择器层面过滤 |
| 点击发送后网络异常 | 不涉及（纯前端，无网络请求） |
| 输入框已有文字时点击语音 | 语音识别文字会替代当前输入框内容 |

### 关联模块

- → AI 意图路由：发送消息触发路由
- → 对话区：插入用户消息气泡

### 不做（本期）

- 真实语音识别 API
- 图片压缩
- 多图发送
- 表情/文件发送
- 输入联想/自动补全

## 6. 空态与页面初始化

### 功能说明

空态是用户首次进入对话页（无历史对话）时展示的引导界面。同时，页面通过 `sessionStorage` 中的 `aiSource` 字段判断用户来源，执行不同的初始化行为。

### 交互流程

**空态**：
1. 页面中间展示 AI 龙老师头像（80×80）+ 标题 + 状态文字
2. 下方展示 4 个快捷提问卡片（百科类问题）
3. 用户点击任意快捷提问 → 隐藏空态 → 自动发送该文字 → 触发 AI 回复

**页面初始化**：

| aiSource 值 | 来源 | 初始化行为 |
|-------------|------|------------|
| `home-focus` | 首页点击输入条 | 自动 focus 输入框弹出键盘，不发送消息 |
| `home-voice` | 首页语音识别完成 | 隐藏空态 → 自动发送识别文字 → 触发 AI 回复 |
| `home-input` | 首页文字输入 | 同上 |
| `home-prompt` | 首页快捷提问 | 同上 |
| `home-image` | 首页拍照上传 | 隐藏空态 → 自动发送图片+文字 → 触发 AI 回复 |
| 无 / 其他 | 直接进入 | 显示空态 |

### 业务规则

- 初始化完成后立即清除 `sessionStorage` 中的 `aiQuestion`、`aiSource`、`aiImage`，避免刷新页面时重复发送
- 快捷提问与首页 AI 卡片轮播中的百科问题保持一致
- 空态在第一条消息发送后隐藏，不可恢复

### 数据说明

> **快捷提问列表**：4 条百科问题（天空蓝色、恐龙灭绝、光速、人体骨头），前端硬编码。

### 异常场景

| 场景 | 处理方式 |
|------|---------|
| sessionStorage 中有 aiSource 但 aiQuestion 为空 | `home-focus` 来源正常（只弹键盘）；其他来源使用默认文字"请帮我看看这道题" |
| sessionStorage 解析异常 | try-catch 包裹，异常时显示空态 |
| aiSource=home-image 但 aiImage 为空 | 使用默认文字发送，不含图片 |

### 关联模块

- ← `首页-v3.html`：通过 sessionStorage 传递用户意图和内容
- → 对话区：初始化时自动插入消息

### 不做（本期）

- 历史对话恢复
- 空态的个性化推荐（如基于学习进度推荐提问）
- 多个对话会话管理

## 页面跳转关系

```
对话.html
├── 作业/作业批改.html?show=result ........... AI 批改结果卡片
├── 作业/作业批改.html?mode=qa&show=result ... AI 搜题结果卡片
├── 作业/作业批改.html?mode=wrong&show=result  录入错题卡片
├── 错题本/错题本.html ...................... 错题本卡片
├── 试卷/试卷擦除.html?show=result .......... 试卷擦除卡片
├── 错因诊断/错因诊断.html?show=result ....... 错因诊断卡片
└── ← 返回（history.back → 首页）
```
