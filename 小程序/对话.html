对话页 产品稿

版本：v1.1 · 2026-04-15

1. AI 意图路由

功能说明

对话页的核心逻辑——AI 接收用户输入后，按优先级匹配意图，决定返回文字回复还是工具功能卡片。

选择规则路由而非大模型自由生成，是因为 MVP 阶段需要确定性的行为来验证产品逻辑，后续版本可替换为真实 NLU。

功能边界：只做意图匹配和回复分发，不做真实 AI 推理。

交互流程

AI 按以下优先级逐层匹配，命中即停止：

优先级意图触发条件AI 回复
1追问上一轮返回了功能卡片，且用户输入含追问词上下文相关的详细分析
2加入错题本同时包含"加入/存/收录/放到/添加" + "错题"有上下文：确认已加入；无上下文：提示先拍照批改
3工具类包含工具关键词（见下表）对应工具的功能卡片
4图片消息用户发了图片但未命中上述意图"让我看看这道题…" + 苏格拉底引导
5百科问答命中 9 组预设百科 QA详细知识回答
6兜底以上都未命中通用引导回复

工具关键词映射

关键词返回卡片跳转目标
批改、检查作业批改报告批改结果页
擦除、还原、空白试卷擦除擦除结果页
错因、诊断、薄弱、分析错因报告错因诊断页
错题 + 上传/录入录入确认录错题结果页
错题本、错题错题本概览错题本列表页
搜题 / 图片+（这道题/怎么做/看看）搜题结果 + 苏格拉底引导搜题结果页

业务规则

  • 功能卡片上的跳转直接进入工具结果页（跳过拍照/上传步骤），因为用户在对话中已提供了图片或文字描述
  • 每次返回卡片后记录卡片类型，用于追问检测
  • 多个工具关键词同时命中时，按上表顺序取第一个

数据说明

百科知识库：9 组 QA，覆盖天空蓝色、恐龙灭绝、人体骨头、光速、地球大小、月球、下雨、太阳等。每条回答包含数据和比喻，适合中小学生阅读。
功能卡片结构：图标 + 标题 + 副标题 + 预览行（✓/✗/📊）+ "点击查看"跳转按钮。

异常场景

场景处理方式
用户输入为空或只有空格发送按钮不可点击
百科问答与工具关键词同时命中工具优先级更高

关联模块

  • ← 首页：接收用户来源和意图
  • → 各工具结果页：功能卡片点击跳转
  • ↔ 苏格拉底引导：搜题意图触发
  • ↔ 追问处理：功能卡片后的追问

不做（本期）

  • 真实 NLU 意图识别
  • 多轮对话上下文理解（仅一轮追问）
  • 工具优先级动态调整

2. 苏格拉底式引导

功能说明

搜题场景下，AI 不直接给答案，而是通过 5 步引导帮助学生自主思考。核心教育理念——"教会思考方法比给答案更重要"。

固定 5 步而非动态步数，是因为 MVP 阶段需要可预测的交互流程。后续版本可结合大模型做动态引导。

交互流程

步骤AI 说目的
1"你先看看这道题，它考的是哪个知识点？"定位知识点
2"那你觉得解这道题的第一步应该做什么？"引导解题思路
3"试试把已知条件列出来，找找它们之间的关系"条件分析
4"你的方向对了！继续算下去，写出你的过程"鼓励自主完成
5"太棒了！你自己想出来的。要不要再练一道？"正向反馈

业务规则

  • 用户每次回复推进一步（不分析内容）
  • 5 步完成后引导结束，后续回复进入正常路由
  • 引导过程中用户发送新的工具关键词，中断引导进入新路由
  • 引导状态不持久化，刷新页面后从头开始

异常场景

场景处理方式
用户快速连发多条消息每条推进一步，可能快速跳完
引导过程中刷新页面引导重置

不做（本期）

  • 根据学生回答智能调整引导方向
  • 引导完成后自动出练习题

3. 追问处理

功能说明

用户看到功能卡片后的自然追问——如看到批改结果后问"第 2 题为什么错"，AI 根据上一轮卡片类型给出上下文相关的具体回复。

交互流程

上一轮卡片追问类型AI 回复
批改结果"第X题为什么错"该题的题目、错误答案、正确答案和错因
批改结果"为什么错/错在哪"汇总全部错题的不同错因
错因诊断"计算类"细分计算类错误的子类型和占比
错因诊断"出题/练习"返回针对薄弱点的练习题
搜题结果任意追问继续苏格拉底引导

业务规则

  • 追问检测关键词：第X题、为什么、怎么错、错在、讲一下、不懂、不会、再看看、详细
  • 仅支持一轮追问——追问后不更新卡片类型，再次输入进入正常路由
  • 追问回复为纯文字，不返回新卡片

异常场景

场景处理方式
追问词和工具词同时命中追问优先

不做（本期）

  • 多轮追问
  • 追问后返回新卡片

4. 对话区 UI

功能说明

承载所有消息展示，包括用户消息、AI 文字回复、AI 功能卡片、输入中指示器。参考豆包、Gemini 的对话 UI。

交互流程

1. 用户发送文字：右侧气泡，文字自适应宽度

2. 用户发送图片：右侧气泡 + 图片缩略图，上方可附带文字

3. AI 文字回复：先展示三点跳动指示器（模拟思考），然后替换为文字气泡

4. AI 功能卡片：同上，先指示器后展示卡片

5. 点击功能卡片"点击查看"按钮：跳转对应工具结果页

业务规则

  • 新消息后自动滚动到底部
  • 功能卡片预览行最多 3 行

异常场景

场景处理方式
刷新页面对话记录丢失，回到空态
连续快速发送按顺序依次处理，每条消息等前一条 AI 回复完成后再处理下一条
AI 回复加载中对话区展示三点跳动指示器（左侧，AI 头像旁），期间发送按钮可继续点击但消息排队
AI 回复失败（网络异常）在 AI 消息位置显示「回复失败」红色提示 + 「重试」按钮，点击重新发送最后一条用户消息
图片发送失败图片缩略图左下角显示红色感叹号，点击弹出「重新发送」选项

不做（本期）

  • 对话记录持久化
  • 消息撤回/删除
  • 流式打字机效果
  • 消息时间戳
  • Markdown 渲染

5. 输入栏

功能说明

底部固定输入栏，四个元素：相机按钮、文字输入框、语音按钮、发送按钮。

交互流程

1. 文字发送：输入内容 → 发送按钮变为可点击 → 点击发送或回车 → 发送消息 → 输入框清空

2. 图片发送：点击相机 → 选择图片 → 缩略图预览出现在输入框上方（可移除） → 点击发送 → 发送含图片的消息

3. 语音/文字模式切换：点击左侧麦克风/键盘图标 → toggle 切换输入模式

4. 语音发送：长按"按住说话"（≥500ms）→ 弹出录音浮层 → 松开后识别 → 识别完毕自动发送

业务规则

  • 发送按钮：输入框有内容或有待发送图片时可点击，否则禁用
  • 回车发送，Shift+回车换行
  • 文字输入框自适应高度，有最大高度限制

异常场景

场景处理方式
选择超大图片本期不做大小校验，后续版本增加压缩和大小限制

不做（本期）

  • 真实语音识别
  • 图片压缩
  • 多图发送
  • 表情/文件发送

6. 空态与页面初始化

功能说明

空态是无历史对话时展示的引导界面。页面根据进入来源执行不同初始化行为。

交互流程

空态：页面居中展示 AI 龙老师圆形头像（80px，紫色渐变边框）+ 标题「AI 龙老师」（16px bold）+ 副标题「你的专属学习助手」（13px #999）+ 下方 2×2 网格排列的 4 个快捷提问卡片（白底圆角卡片，每张含 emoji 图标 + 提问文字）。点击任意快捷提问 → 隐藏空态 → 自动发送该问题文字 → AI 回复。

来源处理：

来源初始化行为
首页点击输入条弹出键盘，不发送消息
首页语音识别完成自动发送识别文字
首页短按"按住说话"进入语音模式（不弹键盘，底部显示"按住说话"），不发送消息
首页快捷提问自动发送提问文字
首页拍照上传自动发送图片+文字
直接进入显示空态

业务规则

  • 来源信息消费后立即清除，避免刷新时重复发送
  • 快捷提问与首页轮播中的百科问题保持一致
  • 空态在第一条消息发送后隐藏

异常场景

场景处理方式
来源信息丢失显示空态，用户手动输入

不做（本期）

  • 历史对话恢复
  • 空态个性化推荐
  • 多对话会话管理
