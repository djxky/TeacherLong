1. 页面概述

功能说明

错因诊断是一个单题拍照诊断功能。用户对准一道题目拍照，AI 进行深度分析后给出全链路错因诊断结果。

与单纯的"对错判断"不同，错因诊断深入到解题思维链路，按"审题感知→知识唤醒→推演求解"等维度逐一分析，帮助学生和家长理解错在哪、为什么错、怎么改。

支持两种入口：拍照诊断（相机拍一道题）和错题本单题诊断。

功能边界：本页面只展示 AI 诊断结果，不支持在线修改答案或重新作答。

学科支持范围

当前仅开放：数学。后续视 AI 能力和准确率逐步扩展到其他学科。

整体流程

入口 A · 拍照诊断：相机拍照 → AI 分析中（约 20 秒）→ 诊断结果页（图片背景 + 底部弹窗）

如果 AI 未能在题库中匹配到该题目 → 显示"未搜到题目"页面

入口 B · 错题本诊断：错题本详情页点击诊断按钮 → 已诊断则直接展示结果 / 未诊断则先 AI 分析（约 20 秒）→ 单题诊断页（卡片纵向布局）

2. 拍照入口（相机页）

功能说明

在作业批改页面的"错因诊断"模式下拍照，相机聚焦框变为单题对焦框（居中小矩形），引导用户对准一道题目拍照。

交互流程

1. 进入：在作业批改相机页面底部 Tab 切换到"错因诊断"

2. 对焦：屏幕中央出现绿色单题聚焦框（约 75% 宽 × 140px 高），框外半透明遮罩

  • 聚焦框下方第一行：「对准一道题目，拍照诊断错因」（白色 12px）
  • 聚焦框下方第二行：「当前仅支持数学题目」（白色半透明 10px）

3. 拍照：点击快门 → 进入扫描动画 → 提示"拍照成功，正在分析..." → 跳转到错因诊断页面

4. 关闭：点击左上角关闭按钮 → 返回上一页面（首页）

业务规则

  • 一次只拍一道题目，不支持整页拍照
  • 拍照后自动进入分析流程，无需手动确认
  • 相机页面需提示用户当前仅支持数学，避免拍其他学科后因"未搜到"而困惑

3. AI 分析中页面

功能说明

拍照后进入的等待页面，展示用户拍摄的图片和 AI 分析进度动画，分析耗时约 20 秒。

页面结构

1. 顶部导航栏：返回箭头 + 标题"错因诊断"

2. 拍照图片预览：圆角卡片展示用户拍摄的题目照片

3. 不确定进度环：绿色弧线持续旋转的加载动画，中心展示脑袋 emoji 的脉冲动画，不展示百分比和倒计时

4. 状态文字：轮播展示分析进度文案，带淡入淡出过渡效果

5. AI 思考卡片：底部卡片展示"AI 龙老师正在深度分析" + 跳动圆点 + shimmer 光效

状态文字轮播

按分析进度均匀切换以下文案：

  • 正在理解题目内容...
  • 分析作答过程...
  • AI 深度思考中...
  • 诊断错误原因...
  • 生成学习建议...

业务规则

  • 不展示具体的百分比进度和剩余时间，因为实际分析耗时不确定
  • 分析过程中用户可以点击返回按钮取消，返回相机页面
  • 文案通用于文科和理科，不涉及具体学科步骤

分析结果分支

AI 分析完成后根据结果进入不同页面：

结果跳转目标过渡方式
成功匹配题目并完成诊断诊断结果页（第 4 节）loading 消失 → 结果页滑入
未能在题库中匹配到题目未搜到题目页（第 5 节）loading 消失 → 未搜到页展示

说明：搜题和诊断在后端同一流程中完成，用户侧不感知中间环节，只看到 loading → 最终结果。

异常场景

场景占位图主文案副文案操作
分析超时（超过 60 秒）沙漏图标 · 48px 灰底圆分析超时请稍后再试按钮「重新拍照」→ 返回相机
网络异常断网图标 · 48px 灰底圆网络连接失败请检查网络后重试按钮「重新拍照」→ 返回相机
用户中途返回———取消分析，返回相机页面

4. 诊断结果页

功能说明

分析完成后展示的结果页面，采用图片背景 + 底部滑动弹窗的交互模式（类似拍照解题）。背景全屏展示用户拍摄的题目图片，底部弹窗承载诊断内容。

页面结构

1. 图片背景层：深色背景全屏展示用户拍摄的题目照片

2. 顶部导航栏：半透明渐变遮罩上叠加白色返回箭头 + 白色标题"错因诊断"

3. 底部诊断弹窗：从底部滑入，占屏幕 70% 高度，圆角 24px

底部弹窗内容（从上到下）

1. 拖拽手柄：灰色小条（40×5px），点击可收起/展开弹窗

2. AI 错因摘要：一段概括性文字，核心错误点以黄底加粗高亮标注

3. 全链路错因诊断：按解题链路维度逐一分析

  • 每个维度标记"掌握 ✓"（绿色）或"待掌握 ✕"（红色）
  • 掌握维度：展示分析说明（灰色文字）
  • 待掌握维度：展示错误详情（红色文字）+ 深度归因分析（绿色文字）

4. 举一反三建议：总结文字 + 2 个 Tip 卡片

5. 底部操作栏：固定在弹窗底部，包含两个按钮

  • 「加入错题本」：次要按钮，点击后变为"已加入"不可再次点击
  • 「再拍一题」：主按钮，返回相机页面

交互细节

1. 弹窗以 0.4s 弹性动画从底部滑入（分析完成后 100ms 延迟触发）

2. 弹窗内容区域可上下滚动，底部操作栏固定不动

3. 点击拖拽手柄可收起弹窗，露出完整的题目照片

4. 点击返回按钮 → 返回相机页面（history.back）

诊断数据结构

字段说明示例
错因摘要一段概括性文字，核心错误点黄底高亮"误判负号运算规则"
诊断维度按解题链路逐一分析，每个维度含名称、子描述、掌握状态审题感知 / 知识唤醒 / 推演求解
错误详情待掌握维度的具体错误描述（红色）"符号出错，错误地写成了…"
深度归因待掌握维度的根因分析 + 正确推演（绿色）"忽略了负号的运算规则…"
举一反三总结文字 + 2 个 Tip 卡片"先确定符号再计算数值"

业务规则

  • 每次诊断只针对一道题目，不存在题目切换
  • "加入错题本"为一次性操作，点击后按钮变灰不可再次点击
  • 弹窗内部独立滚动，不影响背景层
  • 诊断维度数量和名称由 AI 动态生成，不固定

异常场景

场景占位图主文案副文案操作
重复点击"加入错题本"———按钮已变"已加入"，不可再次点击

5. 未搜到题目页

功能说明

当 AI 未能在题库中匹配到用户拍摄的题目时，展示此页面，引导用户重新拍照。

页面结构（占位图 + 文案格式）

元素内容
占位图搜索减号图标 · 80px 黄底圆
主文案未搜到相关题目
副文案无法进行错因分析
建议卡片确保拍照清晰 / 对准单道题目 / 题目和作答完整可见
操作按钮「重新拍照」→ 返回相机页面

交互流程

1. AI 分析完成后判定未匹配到题目 → loading 消失，展示未搜到页面

2. 用户阅读拍照建议 → 点击「重新拍照」→ 返回相机页面

3. 点击顶部返回箭头 → 同样返回相机页面

业务规则

  • 未搜到不等于拍照失败，用户拍照是成功的，只是 AI 没有在题库中找到匹配题目
  • 用户拍了非数学学科的题目也会走到此页面（相机页已提示"当前仅支持数学"，但不阻止拍照）
  • 提示文案应避免让用户觉得是自己操作有误，语气偏引导而非指责

6. 单题诊断页（来自错题本）

功能说明

独立全屏页面，从错题本进入，展示单道错题的完整错因诊断。与拍照诊断结果内容一致，但采用卡片式纵向布局（非弹窗模式），更适合沉浸式阅读。

进入分支

从错题本详情页点击诊断按钮进入，携带题目 ID + 是否已分析标记：

场景进入行为用户感知
已生成诊断直接渲染结果页，无 loading点击按钮 → 立即看到诊断结果
未生成诊断先展示 AI 分析中动画（约 20 秒），完成后渲染结果点击按钮 → loading → 诊断结果

交互流程

1. 进入方式：从错题本详情页点击"查看/生成错因诊断"按钮 → 携带题目 ID 参数

2. 浏览：依次展示错题原图、题目内容、AI 错因摘要、全链路诊断、举一反三建议

3. 返回：点击底部"返回错题本"按钮或顶部返回箭头 → 返回错题本页面

业务规则

  • 仅当错题有学生原始作答时，错题本中才显示"错因诊断"入口
  • 仅已开放学科（当前仅数学）展示诊断入口
  • 页面独立于拍照诊断流程，不依赖相机上下文

异常场景

场景占位图主文案副文案操作
题目 ID 无效文件缺失图标 · 48px 灰底圆题目不存在该题目可能已被删除按钮「返回错题本」
无诊断数据诊断图标 · 48px 灰底圆暂无诊断数据该题目还未进行错因诊断按钮「返回错题本」
