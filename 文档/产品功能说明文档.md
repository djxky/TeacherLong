依据《产品稿标注文字规范》撰写，用于页面、组件与功能说明时统一语气与结构。语气为客观说明，用「这里」「这个」「用于」等指代；用词上输入用「填写」、下拉用「选择」、参数用「配置」、复选框用「勾选」。  
**本文档同时描述各页面的「页面跳转逻辑」与「按钮点击效果」，便于产品、开发与测试对照实现。**

## 二、首页（首页.html）

### 2.1 页面框架

- **整页容器**：这是整个首页的最外层容器，可以理解为单页内的多面板布局（首页面板 + 我的面板），也可作为后续扩展 Tab 的复用框架。
- **安全区**：内容区底部预留安全区内边距，用于适配带 Home 指示条的设备，避免内容被遮挡。

### 2.2 页面跳转逻辑（首页）

| 从哪里进入首页 | 说明 |
|----------------|------|
| 登录成功、作业批改页关闭、个人信息页返回、AI 讲题页返回等 | 首页为应用主入口，多处可返回至此。 |

| 从首页可跳转至 | 触发方式 |
|----------------|----------|
| 作业批改页（作业批改模式） | 点击「AI 作业批改」长条卡片。 |
| 作业批改页（拍照答疑模式） | 点击「拍照解题」长条卡片。 |
| 作业批改页（录错题模式） | 点击「上传错题」宫格入口。 |
| 错题本列表页 | 点击「错题本」宫格入口。 |
| 试卷擦除页 | 点击「试卷擦除」宫格入口。 |
| 我的面板（同页内切换） | 点击右上角「我的」图标；或点击底部 Tab「我的」。 |

### 2.3 顶部操作区与按钮点击效果

- **历史记录按钮**：位于首页面板右上角。**点击效果**：当前为弹出 Toast「历史记录（可跳转独立页）」；若已实现历史记录独立页，则改为跳转至该页面。
- **我的按钮**：位于历史记录按钮左侧。**点击效果**：同页内切换到「我的」面板，首页内容隐藏、我的面板展示；底部 Tab「我的」同步高亮。

### 2.4 欢迎区（学伴区）

- **学伴形象**：这里是学伴（龙老师）的展示区域，放置头像或形象图，用于强化「AI 学伴」的陪伴感。
- **问候文案**：这里的标题与副标题可替换为当前用户名称和固定话术，比如「HI 薛云腾同学」「我是龙老师，你的 AI 学伴」，最终会作为首屏的主文案展示。

### 2.5 功能区（Bento 布局）与按钮点击效果

- **AI 作业批改（长条卡片）**  
  - **点击效果**：先写入 `sessionStorage.cameraMode = 'correction'`，再跳转至 `首页/作业批改.html?mode=correction`，作业批改页加载后默认选中「作业批改」模式。  

- **拍照解题（长条卡片）**  
  - **点击效果**：先写入 `sessionStorage.cameraMode = 'qa'`，再跳转至 `首页/作业批改.html?mode=qa`，作业批改页默认选中「拍照答疑」模式。  

- **错题本（宫格入口）**  
  - **点击效果**：跳转至 `错题本/错题本 01.html`。  

- **上传错题（宫格入口）**  
  - **点击效果**：先写入 `sessionStorage.cameraMode = 'wrong'`，再跳转至 `首页/作业批改.html?mode=wrong`，作业批改页默认选中「录错题」模式。  

- **试卷擦除（宫格入口）**  
  - **点击效果**：跳转至 `试卷擦除/试卷擦除.html`。  

- **入口卡片**：每个入口为可点击的卡片组件，用于承载图标与主标题；当前版本不展示副标题，仅保留主标题以保持简洁。点击时有轻微按压动效（如 scale）。

### 2.6 我的面板与按钮点击效果

- **我的面板**：与首页并列的另一个内容面板，展示手机号、学校、班级、姓名等信息及「退出登录」按钮。
- **底部 Tab「首页」**  
  - **点击效果**：同页内切回首页面板，底部 Tab「首页」高亮。  
- **底部 Tab「我的」**  
  - **点击效果**：同页内切换到我的面板，底部 Tab「我的」高亮。  
- **退出登录按钮**  
  - **点击效果**：清除登录相关 session，跳转至 `登录/index.html`。

### 2.7 全局反馈

- **Toast**：用于在操作后给予简短反馈（如「操作成功」「历史记录（可跳转独立页）」），展示在页面顶部中央，约 2 秒后自动消失。

## 三、作业批改 / 拍照页（作业批改.html）

本页面是统一的「拍作业 / 拍题目」入口，通过底部模式切换实现作业批改、拍照答疑、录错题、试卷擦除四种能力。

### 3.1 页面跳转逻辑（作业批改页）

| 从哪里进入本页 | 说明 |
|----------------|------|
| 首页「AI 作业批改」 | 带 `?mode=correction`，默认「作业批改」模式。 |
| 首页「拍照解题」 | 带 `?mode=qa`，默认「拍照答疑」模式。 |
| 首页「上传错题」 | 带 `?mode=wrong`，默认「录错题」模式。 |
| 错题本内「上传错题」 | 跳转 `作业批改.html?mode=wrong`。 |

| 从本页可跳转至 | 触发方式 |
|----------------|----------|
| 首页（首页.html） | 取景态或结果态顶部「关闭」、取景态关闭按钮。 |
| 试卷擦除页 | 当前为「试卷擦除」模式时，拍照/选图并完成扫描态后自动跳转。 |
| 错题本列表页 | 录错题智能识别态下点击「上传」并选择题目后，成功上传后跳转。 |
| AI 讲题页 | 题目详情抽屉内点击「AI 讲解」。 |

### 3.2 页面框架与状态

- **页面容器**：这是相机/拍照流程的最外层容器，内部包含多种状态：取景、扫描中、批改结果、录错题智能识别、题目详情等。状态切换通过显隐与类名控制。
- **URL 与模式**：进入时读取 `?mode=` 或 `sessionStorage.cameraMode`，用于决定初始选中的模式（correction / qa / wrong / erase）。

### 3.3 顶部栏（取景态）与按钮点击效果

- **关闭按钮**（左上角）  
  - **点击效果**：跳转至 `首页/首页.html`。  

- **中部标签**：这里展示「AI 自动识别」等固定文案，无点击。  
- **闪光灯按钮**（右上角）  
  - **点击效果**：切换闪光灯开/关状态；图标在「线框闪电」与「实心闪电」间切换，按钮背景与颜色高亮变化；弹出 Toast「闪光灯开」或「闪光灯关」。  

### 3.4 取景区与取景框

- **预览区**：这里是相机预览或占位图的展示区域，用于模拟/真实显示拍摄画面。取景框与提示文案叠加在该区域上。
- **取景框（作业批改 / 试卷擦除）**：用于整页拍摄场景，框内为有效识别区域。框线为白色半透明，四角有 L 形角标；框内包含九宫格参考线（两横两竖将框均分为九格），用于辅助用户对齐整页、保持题目平行与参考性。
- **取景框（拍照答疑 / 录错题）**：用于单题或局部拍摄，框为横向窄条，带模式对应的边框色（如黄色/橙色），框内有一条水平参考线，用于对准单题。
- **提示文案**：  
  - 作业批改 / 录错题（整页）：这里展示「一次拍摄一整页，题目平行参考性」。  
  - 拍照答疑：这里展示「对准题目，精准答疑」。  
  - 试卷擦除：这里展示「整页拍摄，擦除笔迹」。

### 3.5 底部操作区（取景态）与按钮点击效果

- **模式切换标签**（作业批改 / 拍照答疑 / 录错题 / 试卷擦除）  
  - **点击效果**：当前点击的标签高亮（active），取景框与提示文案切换为对应模式；仅同页内状态变化，不跳转。  

- **相册按钮**（快门左侧）  
  - **点击效果**：调起系统相册（`input[type=file][accept=image/*]`）；用户选图后，取景态隐藏、进入扫描态，提示文案按当前模式展示；约 2 秒后与「拍照」流程一致：作业批改→结果页，拍照答疑→题目详情抽屉，录错题→智能识别页，试卷擦除→跳转试卷擦除页。  

- **快门按钮**  
  - **点击效果**：取景态隐藏、进入扫描态；约 2 秒后：若作业批改→显示批改结果页，若拍照答疑→打开题目详情抽屉，若录错题→显示智能识别页，若试卷擦除→跳转 `试卷擦除/试卷擦除.html`。  

### 3.6 扫描态

- **扫描态容器**：拍照或选图后全屏展示，用于表示「AI 正在处理」。
- **提示文案**：根据当前模式展示「AI 正在识别试卷…」「AI 正在聚焦题目…」「正在加入错题本…」「正在擦除笔迹…」等。
- **扫描动画**：扫描线动画约 2 秒后自动进入下一状态，无需用户点击。

### 3.7 批改结果态与按钮点击效果

- **结果页关闭按钮**（左上角）  
  - **点击效果**：跳转至 `首页/首页.html`。  

- **题目卡片**  
  - **点击效果**：打开题目详情抽屉（`openDetail(题目索引)`），展示该题题目、视频讲解、答案解析及「加入错题本」「AI 讲解」按钮。  

- **再拍一张**  
  - **点击效果**：结果页隐藏，取景态重新显示，可再次拍照。  

### 3.8 题目详情（底部抽屉）与按钮点击效果

- **抽屉顶部拖条或关闭图标**  
  - **点击效果**：关闭抽屉（`closeDetail()`），抽屉下滑消失。  

- **题目编号按钮**（批改模式且多题时）  
  - **点击效果**：切换当前题目索引，抽屉内题目内容、解析、AI 讲解按钮随之刷新。  

- **加入错题本**  
  - **点击效果**：弹出 Toast「已加入错题本」；按钮置灰、文案改为「已加入」，再次点击无效果。  

- **AI 讲解**  
  - **点击效果**：将当前题目与解析写入 sessionStorage，跳转至 `拍照检查作业/ai 讲题.html`。若该题不支持 AI，按钮为禁用态。  

### 3.9 录错题 - 智能识别态与按钮点击效果

- **返回按钮**  
  - **点击效果**：智能识别页隐藏，取景态重新显示。  

- **自动识别框 / 手动框**  
  - **点击效果**：勾选或取消勾选该框（选中态边框与背景高亮）；至少勾选一道题时「上传」按钮变为可点。  

- **手动框题**  
  - **点击效果**：在页面指定位置新增一个可勾选的手动框，点击该框可勾选/取消。  

- **上传**  
  - **点击效果**：未选题目时点击无反应；已选题目时弹出 Toast「已加入错题本，共 N 道」，约 1.2 秒后跳转至 `错题本/错题本 01.html`。  

### 3.10 答疑模式结果

- **单题答疑**：拍照答疑模式下识别完成后直接打开题目详情抽屉，行为与批改模式点击题目卡片一致；「AI 讲解」点击后同样跳转 AI 讲题页。

### 3.11 试卷擦除模式

- **跳转**：在「试卷擦除」模式下，拍照或选图并完成扫描态后，直接跳转至 `试卷擦除/试卷擦除.html`，不留在本页。

## 四、错题本（错题本 01.html）

### 4.1 页面跳转逻辑（错题本）

| 从哪里进入本页 | 说明 |
|----------------|------|
| 首页「错题本」宫格 | 跳转至 `错题本/错题本 01.html`。 |
| 作业批改页「录错题」上传成功 | 上传完成后跳转至本页。 |

| 从本页可跳转至 | 触发方式 |

| 打印错题页 | 底部「打印错题」或更多菜单中「打印错题」。 |
| 题目详情/拍照流程 | 列表内嵌上传错题流程时，按步骤进入拍照→智能识别→上传。 |

### 4.2 页面框架

- **错题本列表页**：这是错题本的主页面，用于展示已收录的错题列表或分组，支持科目筛选、题型筛选、编辑模式等。

### 4.3 顶部栏与按钮点击效果

- **返回/关闭**  
  - **点击效果**：跳转至首页（如 `../首页/index.html`）。  

- **标题**：这里展示「错题本」等固定标题，无点击。  
- **更多按钮**（三点）  
  - **点击效果**：展开下拉菜单，展示「上传错题」「打印错题」；点击菜单外区域可收起菜单。  

### 4.4 更多菜单与按钮点击效果

- **上传错题**  
  - **点击效果**：跳转至 `首页/作业批改.html?mode=wrong`，进入录错题拍照流程；拍完并上传后可再回到错题本。  

- **打印错题**  
  - **点击效果**：跳转至 `错题本/打印错题.html`。  

### 4.5 列表区与按钮点击效果

- **科目 Tab**（如数学、英语）  
  - **点击效果**：切换当前科目，列表与筛选标签随之刷新。  

- **筛选标签**（全部、单选题、填空题、解答题等）  
  - **点击效果**：切换筛选条件，列表只展示符合的错题。  

- **错题卡片**  
  - **点击效果**：非编辑模式下打开该题详情（抽屉或新层）；编辑模式下可配合勾选框多选。  

- **上传错题 / 打印错题**（底部按钮，若保留）  
  - **点击效果**：与更多菜单中「上传错题」「打印错题」一致，分别跳转作业批改页（录错题模式）与打印错题页。  

- **编辑模式、全选、删除**  
  - **点击效果**：进入编辑模式后出现勾选框与「全选」「删除」；全选可勾选当前列表全部；删除可删除已选题目。

## 七、试卷整理（试卷整理.html）

### 7.1 页面跳转逻辑

| 从哪里进入本页 | 说明 |
|----------------|------|

## 十三、全局与通用说明

### 13.1 设备与框架

- **iPhone 机身模拟**：多页面采用固定宽高（如 393×852）的机身框与 Dynamic Island 占位，用于在设计稿或演示中模拟真机效果，实际 H5 可配置为响应式或全屏。
- **安全区**：顶部预留状态栏/刘海区域，底部预留 Home 指示条区域，避免关键按钮与内容被遮挡。

- **作文批改**：写作文后拍给 AI 批改、得点评，可作为首页长条主入口或收进「拍给学伴」统一入口。
- **阅读**：AI 阅读理解等，可放在首页底部快捷入口或独立 Tab，与错题本、上传错题、试卷擦除等并列时，底部可为 2×2 或一行多格布局。
