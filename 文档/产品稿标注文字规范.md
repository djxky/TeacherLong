# 产品稿标注文字规范（参考 9-4）

根据设计稿右侧/指向处的标注风格整理，用于页面、组件、配置说明时统一语气与结构。

---

## 一、整体风格

- **位置**：标注在元素右侧或指向具体控件，用紫色/红色框。
- **语气**：客观、说明性，用「这里」「这个」「用于」等指代清晰。
- **长度**：单句或短段，不啰嗦，必要时用「比如」「等」举例。

---

## 二、常用句式

### 1. 定义/解释（这是什么、能做什么）

- 「这是…，可以理解为…，也可以…」
- 「…组件：一个常用的…组件，可以…」
- 「…弹窗，用于…」
- 「…：用于…，可以配置…」

**示例：**
- 这是整个页面的框架，可以理解为页面的最外层容器，也可以作为全局组件进行复用。
- 卡片组件：一个常用的容器组件，可以放置其他组件在里面。
- 表格组件：用于展示结构化数据，可以配置列、数据、分页等。

### 2. 可替换/可填写内容

- 「这个文本可以替换为…，或者其他…」
- 「这个文本区域可以放置一些补充说明或描述性文字」
- 「…，这里填写…，最终会作为…」
- 「…，这里选择…，比如…」

**示例：**
- 这个文本可以替换为页面标题，或者其他文字。
- 页面名称，这里填写页面的标题或名称，最终会作为路由的名称。
- 页面类型，这里选择页面的类型，比如列表页、详情页、表单页等。

### 3. 区域说明（这里是哪一块）

- 「这里是…」
- 「…：这里是…区域，可以…」

**示例：**
- 这里是页面的主要内容区域。
- 页面内容：这里是页面的内容区域，可以拖拽组件到这里。

### 4. 条件/规则说明（什么时候勾选、怎么填）

- 「…：如果…，则勾选。」
- 「是否必填：如果该参数是必填项，则勾选。」
- 「默认值：如果该参数有默认值，则填写。」

**示例：**
- 需要请求：如果页面需要从后端获取数据，则勾选。需要登录：如果页面需要用户登录才能访问，则勾选。
- 请求参数：这里配置页面需要从 URL 获取的参数，或者需要从上一个页面传递过来的参数。

### 5. 配置项说明（每个字段怎么填）

- 「这里填写…，比如…」
- 「这里填写…，方便理解。」
- 「这里选择…，比如…」
- 「…：控制…是否…」

**示例：**
- 这里填写请求参数的名称，比如 pageSize、pageNum、keyword 等。
- 这里填写参数的描述，方便理解。
- 这里选择参数的类型，比如 string、number、boolean 等。
- 是否显示边框：控制卡片是否显示边框。

## 三、按模块整理的用语

| 模块         | 标注用语要点 |
|--------------|--------------|
| 页面框架     | 「最外层容器」「可复用」「标题/内容/描述」 |
| 新建页面     | 「新建页面弹窗」「请输入页面名称」「取消/确定」 |
| 页面配置     | 「页面名称」「页面类型」「需要请求/需要登录」「请求参数」 |
| 请求参数表   | 「参数名」「描述」「类型」「是否必填」「默认值」「添加参数」 |
| 全局配置     | 「全局布局」「布局方式」「主题色」「顶部菜单+侧边菜单」 |
| 页面内容     | 「搜索组件」「拖拽组件到这里」 |
| 卡片         | 「卡片标题」「卡片内容」「是否显示边框」 |
| 表格         | 「表格标题」「表格数据」「支持 JSON 格式」 |
| 图片         | 「图片组件」 |

---

## 四、用词习惯

- **填写**：用于输入框、表单字段（页面名称、参数名、描述、默认值等）。
- **选择**：用于下拉、类型、布局方式等。
- **配置**：用于参数表、全局设置、请求参数等。
- **勾选**：用于复选框（需要请求、需要登录、是否必填等）。
- **放置 / 替换**：用于文本、内容区域说明。
