📋 产品稿 · 数据口径定义
1顶部 KPI 指标栏▶

数据口径

覆盖学校数：有教师实际登录过的学校总数。口径 = AI 龙老师教师端有登录记录的教师所属学校去重 count。

参与教师数：登录过AI 龙老师教师端的教师去重人数。口径 = AI 龙老师教师端有登录记录的 distinct userId（手机号登录即算）。

参与学生数：在AI 龙老师小程序登录过的学生去重人数。口径 = 小程序有登录记录的 distinct userId（手机号登录即算）。

累计服务次数：所有功能的使用次数汇总。口径 = SUM(作业批改次数 + 拍照答疑次数 + AI答疑次数 + 上传错题次数 + 试卷擦除次数 + AI讲题播放次数 + 错题本使用次数 + 教学动画生成次数 + AI命题次数 + AI组题次数)。

累计学习时长：学生使用各功能的累计时间（单位：小时）。口径 = SUM(每次功能使用的 duration_seconds) / 3600，含 AI 讲题播放时长、答疑对话时长、作业批改停留时长。

交互逻辑

页面加载时，五项 KPI 按当前学期时间范围查询并以动画递增展示。

所有指标均为产品自采集数据，不依赖外部教务系统。
2每日活跃用户（DAU）▶

数据口径

学生 DAU：AI 龙老师小程序当日登录去重 UV。口径 = 当日在小程序有登录行为的 distinct userId，按自然日（0:00-23:59）统计。

教师 DAU：AI 龙老师教师端当日登录去重 UV。口径 = 当日在AI 龙老师教师端有登录行为的 distinct userId，按自然日统计。

头部数值：展示「昨日」DAU 值。

交互逻辑

默认选中「学生」Tab。点击切换时，数值、折线颜色同步变化。hover 折线显示当天具体 DAU 值。

数据特征：工作日高、周末低的锯齿状，符合学生和教师的真实使用节奏。
3各年级活跃用户分布▶

数据口径

活跃用户数：最近 7 日（含昨日在内的连续 7 天）在该年级有使用记录的学生去重 UV。口径 = 按用户的年级字段分组，统计近 7 日 distinct userId（使用次数 ≥ 1）。

年级标签：一年级~九年级 + 高一、高二、高三，共 12 个年级。
颜色分段：绿色 = 小学（一~六年级）、青色 = 初中（七~九年级）、紫色 = 高中（高一~高三）。
7 日窗口兼顾数据量充足与时效性，避免单日数据波动过大，也不会像累计口径那样分布长期固化。预期初中段用户最多，小学低年级偏低属正常分布。
4学校覆盖地图▶

数据口径

气泡大小：由于学生暂无学校属性，地图气泡目前仅按该校活跃教师数决定。公式 = 4 + 活跃教师数 × 0.08（px）。

Tooltip · 活跃教师：该校本学期有使用记录的教师去重 UV。
Tooltip · AI创作数：该校教师使用 AI 助教功能（教学动画 / AI命题 / AI组题）的总创作件数。

实时动态弹幕

地图左下角纵向滚动，展示师生近期使用动态，营造「系统正在被使用」的真实感。

内容模板（共 2 大类）：

① 教师创作类
格式：{学校全称} {姓}老师 {动作}【{学科}】{功能名}
　示例：龙城高级中学 张老师 生成了【数学】AI教学动画
　动作枚举：生成了 / 发布了 / 使用
　功能名枚举：AI教学动画 / AI命题 / AI组题试卷 / AI教案备课

② 学生使用类（因学生无学校信息，展示年级+手机尾号）
　格式：{年级}用户 {手机尾号后四位} {动作}
　示例：七年级用户 8273 完成了AI作业批改
　动作枚举：完成了AI作业批改 / 使用了拍照答疑 / 上传了N道错题 / 完成了试卷擦除重做 / 使用了AI答疑
数据口径：
数据来源 = 各功能的操作日志流，取最近 100 条事件。
去重规则 = 同一用户 + 同一功能，30 分钟内合并为 1 条。
排序 = 按事件时间倒序。
不显示时间，避免 demo 演示时暴露静态数据。
显示规则：
滚动方式 = CSS animation 纵向循环滚动，速度 50s/轮。
hover = 暂停滚动。
条目上限 = 最多展示 20 条。
每条格式 = ● {学校全称/手机尾号}（绿色加粗） + 事件描述。教师条目显示学校全称，学生条目显示手机尾号后四位。
V1.0 使用静态模拟数据；V2.0 接入服务端接口轮询（30s 间隔）或 WebSocket 实时推送。
5热门功能 TOP 5▶

数据口径

统计维度：最近 7 日（含昨日在内的连续 7 天）各 AI 功能的使用人数 UV（助学 + 助教两端）。
功能枚举：作业批改 / 拍照答疑 / AI答疑 / AI教学动画 / 上传错题 / AI命题 / 错题本 / 试卷擦除 / AI组题 / AI讲题。

展示方式：取 TOP 5 功能，横向一排 5 张图标卡片 = 功能图标 + 排名徽章（TOP 1~5） + 功能名 + UV 数值。一屏无滚动。
徽章样式：金(1) / 银(2) / 铜(3) / 灰(4-5)。
排序规则：按最近 7 日使用人数 UV 降序。

口径区分

热门 TOP 5 = 最近 7 日 UV（使用人数），体现当前热度，排名每日滚动更新。
右栏助学/助教 = 学期累计 PV（使用次数），体现整体服务量。

两者维度不同，互为补充：TOP 5 回答"最近什么火"，右栏回答"总量做了多少"。
7 日滚动窗口让排名有波动，反映当前热度，同时避免单日数据波动过大。
6AI 助学 · 学生端▶

数据口径

核心指标（3 列彩色卡片）：
• 作业批改 — 学生拍照提交批改的累计次数。口径 = event_name='homework_check' 的 count。
• 拍照答疑 — 学生拍照提问的累计次数。口径 = event_name='photo_qa' 的 count。
• AI 答疑 — 学生发起文本对话答疑的累计次数。口径 = event_name='ai_qa' 的 count。

次要指标（2×2 网格）：
• 上传错题 — 学生主动拍照采集错题的累计次数。口径 = event_name='upload_wrong' 的 count。
• 试卷擦除 — 试卷擦除重做功能使用的累计次数。口径 = event_name='paper_erase' 的 count。
• AI 讲题播放 — AI 视频讲解的累计播放次数。口径 = event_name='ai_explain_play' 的 count。
• 错题本 — 使用错题本功能的累计次数。口径 = event_name='wrong_book' 的 count。
所有数字为本学期累计使用次数（PV），与热门 TOP 5 的最近 7 日 UV（使用人数）口径不同，分别体现「总服务量」和「当前热度」。
7AI 助教 · 教师端▶

数据口径

核心指标（3 列彩色卡片）：
• 教学动画 — 教师使用 AI 生成教学动画的累计数量。口径 = AI 龙老师教师端 content_type='animation' 的创作记录 count。
• AI 命题 — 教师使用 AI 辅助出题的累计次数。口径 = content_type='question' 的 count。
• AI 组题 — 教师使用 AI 自动组卷的累计份数。口径 = content_type='paper' 的 count。

教师创作榜 TOP7

排名指标：教师创作内容总数。口径 = 该教师在AI 龙老师教师端的所有创作记录 count（含动画、命题、组题）。
展示字段：圆形头像、姓名、学校·学科（同一行）、创作数。
头像来源：默认随机头像（DiceBear API），后续接入教师真实头像。
核心指标为本学期累计创作件数（PV），与热门 TOP 5 的最近 7 日 UV 口径不同。教师榜可用于区级会议表彰。
