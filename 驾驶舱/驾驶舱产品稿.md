# 龙老师数据驾驶舱 · 产品稿

> 面向龙岗区教育局领导，一屏展示 AI"龙老师"整体使用成果。  
> 对应页面：`龙老师数据驾驶舱-demo.html`  
> 更新日期：2026-03-19

---

## 一、页面概述

三栏布局，一屏展示，无需滚动：

| 区域 | 位置 | 内容 |
|------|------|------|
| 顶部 KPI 栏 | 全宽横排 | 5 项核心指标 |
| 左栏 | 23% 宽 | 日活趋势折线图 + 年级分布柱状图 |
| 中栏 | 自适应 | 龙岗区学校地图 + 实时弹幕 + 热门功能 TOP 5 |
| 右栏 | 28% 宽 | AI 助学数据卡 + AI 助教数据卡 + 教师创作榜 |

当前无筛选维度（学期切换已移除），页面展示当前学期数据。

---

## 二、顶部 KPI 指标栏

五项核心指标横向排列，页面加载时从 0 动画递增到目标值。

| 指标 | 含义 | 数据口径 |
|------|------|----------|
| **累计覆盖学校** | 有教师实际登录过的学校总数 | 龙老师教师端有登录记录的教师所属学校去重 count |
| **累计参与教师** | 登录过龙老师教师端的教师去重人数 | 龙老师教师端有登录记录的 distinct userId（手机号登录即算） |
| **累计参与学生** | 在龙老师小程序登录过的学生去重人数 | 小程序有登录记录的 distinct userId（手机号登录即算） |
| **累计服务次数** | 所有功能的使用次数汇总 | SUM(作业批改 + 拍照答疑 + AI答疑 + 上传错题 + 试卷擦除 + AI讲题播放 + 错题本 + 教学动画 + AI命题 + AI组题) |
| **累计学习时长** | 学生使用各功能的累计时间（小时） | SUM(每次功能使用的 duration_seconds) / 3600，含讲题播放、答疑对话、批改停留时长 |

**交互逻辑**：页面加载时，五项 KPI 按当前学期时间范围查询并以动画递增展示。

> 所有指标均为产品自采集数据，不依赖外部教务系统。

## 三、左栏

### 3.1 每日活跃用户（DAU 折线图）

面积填充折线图，展示近 30 天日活趋势。支持「学生 / 教师」Tab 切换。

| 指标 | 数据口径 |
|------|----------|
| **学生 DAU** | 小程序当日登录去重 UV = 当日有登录行为的 distinct userId，按自然日（0:00-23:59）统计 |
| **教师 DAU** | 龙老师教师端当日登录去重 UV = 当日有登录行为的 distinct userId，按自然日统计 |
| **头部数值** | 展示「昨日」DAU 值 |

**交互逻辑**：
- 默认选中「学生」Tab
- 点击切换时，数值、折线颜色同步变化
- hover 折线显示当天具体 DAU 值

> 数据特征：工作日高、周末低的锯齿状，符合师生真实使用节奏。

### 3.2 各年级活跃用户分布（横向柱状图 · 最近 7 日）

| 指标 | 数据口径 |
|------|----------|
| **活跃用户数** | 最近 7 日（含昨日在内的连续 7 天）在该年级有使用记录的学生去重 UV = 按用户的年级字段分组，统计近 7 日 distinct userId（使用次数 ≥ 1） |

**年级标签**：一年级~九年级 + 高一、高二、高三，共 12 个年级。

**颜色分段**：
- 绿色 = 小学（一~六年级）
- 青色 = 初中（七~九年级）
- 紫色 = 高中（高一~高三）

> 7 日窗口兼顾数据量充足与时效性，避免单日数据波动过大，也不会像累计口径那样分布长期固化。预期初中段用户最多，小学低年级偏低属正常分布。

## 四、中栏

### 4.1 学校覆盖地图

ECharts 地图渲染龙岗区边界，叠加气泡标注学校位置。由于学生数据暂无学校字段，地图气泡目前**仅按教师活动**进行展示。

| 指标 | 数据口径 |
|------|----------|
| **气泡大小** | 按该校**活跃教师数**决定。公式 = 4 + 活跃教师数 × 0.08（px） |
| **Tooltip · 活跃教师** | 该校本学期有使用记录的教师去重 UV |
| **Tooltip · AI创作数** | 该校教师使用 AI 助教功能（教学动画 / AI命题 / AI组题）的总创作件数 |

**技术方案**：GeoJSON 优先远程拉取阿里 DataV 数据（440307.json），请求失败回退内联备用数据。

### 4.2 实时动态弹幕

地图左下角纵向滚动，展示师生近期使用动态，营造「系统正在被使用」的真实感。

#### 内容模板（共 2 大类）

| 类型 | 格式模板 | 示例 |
|------|----------|------|
| ① 教师创作 | {学校全称} {姓}老师 {动作}【{学科}】{功能名} | 龙城高级中学 张老师 生成了【数学】AI教学动画 |
| ② 学生使用 | {年级}用户 {手机尾号后四位} {动作} | 七年级用户 8273 完成了AI作业批改 |

> 学生数据暂无学校信息，因此使用「年级 + 手机尾号后四位」标识学生。

功能名枚举：
- 教师创作类：AI教学动画 / AI命题 / AI组题试卷 / AI教案备课
- 学生使用类：完成了AI作业批改 / 使用了拍照答疑 / 上传了N道错题 / 完成了试卷擦除重做 / 使用了AI答疑

#### 数据口径

| 规则 | 说明 |
|------|------|
| 数据来源 | 各功能的操作日志流，取最近 100 条事件（去重后） |
| 去重规则 | 同一用户 + 同一功能，30 分钟内合并为 1 条 |
| 排序 | 按事件时间倒序 |

> 弹幕不显示时间，避免 demo 演示时暴露静态数据。

#### 显示规则

| 规则 | 说明 |
|------|------|
| 滚动方式 | CSS animation 纵向循环滚动，速度 50s/轮 |
| hover | 暂停滚动 |
| 条目上限 | 最多展示 20 条 |
| 每条格式 | ● {学校全称/手机尾号}（绿色加粗） + 事件描述。教师条目显示学校全称，学生条目显示手机尾号后四位 |

> V1.0 使用静态模拟数据；V2.0 接入服务端接口轮询（30s 间隔）或 WebSocket 实时推送。

### 4.3 热门功能 TOP 5（最近 7 日）

取**最近 7 日**（含昨日在内的连续 7 天）使用人数（UV）最高的 5 个 AI 功能，横向一排 5 张图标卡片展示，一屏呈现无需滚动。

| 指标 | 数据口径 |
|------|----------|
| **统计维度** | 最近 7 日各 AI 功能的使用人数 UV（助学 + 助教两端） |
| **功能枚举** | 作业批改 / 拍照答疑 / AI答疑 / AI教学动画 / 上传错题 / AI命题 / 错题本 / 试卷擦除 / AI组题 / AI讲题 |
| **排序规则** | 按最近 7 日使用人数 UV 降序，取 TOP 5 |
| **展示方式** | 每张卡片 = 功能图标 + TOP N 排名徽章 + 功能名 + UV 数值 |
| **徽章样式** | 金(1) / 银(2) / 铜(3) / 灰(4-5) |

**与右栏口径区分**：
- 热门 TOP 5 = **最近 7 日 UV**（使用人数），体现当前热度，排名每日滚动更新
- 右栏助学/助教 = **学期累计 PV**（使用次数），体现整体服务量

> 7 日滚动窗口让排名有波动，反映当前真实热度，同时避免单日数据波动过大。

## 五、右栏

### 5.1 AI 助学 · 学生端（龙老师小程序）

分核心与次要两层展示。

#### 核心指标（3 列彩色卡片）

| 指标 | 含义 | 数据口径 |
|------|------|----------|
| **作业批改** | 学生拍照提交批改的累计次数 | event_name='homework_check' 的 count |
| **拍照答疑** | 学生拍照提问的累计次数 | event_name='photo_qa' 的 count |
| **AI 答疑** | 学生发起文本对话答疑的累计次数 | event_name='ai_qa' 的 count |

#### 次要指标（2×2 网格）

| 指标 | 含义 | 数据口径 |
|------|------|----------|
| **上传错题** | 学生主动拍照采集错题的累计次数 | event_name='upload_wrong' 的 count |
| **试卷擦除** | 试卷擦除重做功能的累计使用次数 | event_name='paper_erase' 的 count |
| **AI 讲题播放** | AI 视频讲解的累计播放次数 | event_name='ai_explain_play' 的 count |
| **错题本** | 使用错题本功能的累计次数 | event_name='wrong_book' 的 count |

> 所有数字为本学期累计使用次数（PV），与热门 TOP 5 的最近 7 日 UV（使用人数）口径不同，分别体现「总服务量」和「当前热度」。

### 5.2 AI 助教 · 教师端（龙老师教师端）

#### 核心指标（3 列彩色卡片）

| 指标 | 含义 | 数据口径 |
|------|------|----------|
| **教学动画** | 教师使用 AI 生成教学动画的累计数量 | 龙老师教师端 content_type='animation' 的创作记录 count |
| **AI 命题** | 教师使用 AI 辅助出题的累计次数 | content_type='question' 的 count |
| **AI 组题** | 教师使用 AI 自动组卷的累计份数 | content_type='paper' 的 count |

#### 教师创作榜 TOP7

| 规则 | 说明 |
|------|------|
| **排名指标** | 教师创作内容总数 = 该教师在龙老师教师端所有创作记录 count（含动画、命题、组题） |
| **展示字段** | 圆形头像、姓名、学校·学科（同一行）、创作数 |
| **头像来源** | 默认随机头像（DiceBear API），后续接入教师真实头像 |
| **学校名称** | 使用龙岗区真实学校名称 |
| **视觉规范** | 前三名用金/银/铜色徽章，4-7 名灰色底 |

> 核心指标为本学期累计创作件数（PV），与热门 TOP 5 的最近 7 日 UV 口径不同。教师榜可用于区级会议表彰。

## 六、自适应方案

CSS 变量 `--s` 作为全局缩放因子，通过 `@media` 断点自动调整：

| 屏幕宽度 | --s 值 | 适用场景 |
|----------|--------|----------|
| < 1440px | 1.0 | 笔记本 |
| ≥ 1440px | 1.15 | 大笔记本 |
| ≥ 1680px | 1.3 | 27 寸显示器 |
| ≥ 1920px | 1.45 | 1080P 大屏 |
| ≥ 2560px | 1.8 | 2K/4K 大屏投屏 |

所有 font-size、padding、gap、border-radius 均使用 `calc(Xpx * var(--s))` 实现等比缩放。
